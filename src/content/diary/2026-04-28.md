Followed up on yesterday’s authentication flow. LibreChat uses its own JWT while our SSO depends on Keycloak, so token expiry and unauthorized responses are expected edge cases. Added test cases for the chat store APIs to validate authentication behavior and improve reliability.

Spent time investigating Keycloak sync issues and implemented a retry flow — on unauthorized response, use refresh token to get a new access token and retry the API call. This stabilizes user requests without introducing extra infrastructure complexity. We intentionally avoided adding Celery/Redis to keep the system simpler for this use case.
