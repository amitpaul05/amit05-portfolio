Today I spent a good amount of time tracing a migration mismatch in LiteLLM in our ongoing AYO project. The interesting part was the issue was not Prisma itself — it was the Docker build pipeline silently excluding migrations from our fork. I fixed it at the container layer instead of applying manual DB patches.

Then moved on making delete sync more reliable between LibreChat and Django. Added background sync flow, retry/token refresh helpers, and proper isolation so failures never affect the main delete request.

Then caught a production infra mistake on one of our dev site called Govara which was deployed by me. The app was accidentally running in Next.js dev mode on the server, which explained the unusual RAM spikes. Switched it to a proper production build/start setup and the server stabilized.
