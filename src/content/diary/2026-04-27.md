Today I worked on moving chat storage responsibility from LiteLLM to LibreChat and deployed the change.

Before this, conversations from AYO chat were saved by LiteLLM hook. It was sitting in between the student and the AI, so debugging was harder and sometimes data was saved at the wrong time. I moved this logic directly into LibreChat so that the chat application itself saves conversations after the AI finishes responding.

Now when a student finishes a conversation, LibreChat sends the chat to AYO dashboard immediately using the user’s existing session. No extra token or authentication flow is needed. Also when LibreChat generates a title, it updates the same conversation title in dashboard at the same time. I disabled the LiteLLM hook to avoid double-saving but kept the code for fallback.

This simplified the architecture, reduced dependency on middleware, and made chat storage more reliable. Then I deployed the changes and verified conversations are saved correctly end-to-end.

Later I worked on storing media files for the dashboard. Added models and APIs for handling uploaded media and integrated it with LibreChat so files can be saved and referenced in conversations. This required aligning backend schema with LibreChat payload structure and making sure uploads are persisted correctly. 

After the focused day in office, I spent time revising graph concepts from Grokking Algorithms. Reviewed BFS vs DFS, graph traversal patterns, and when to model problems as graphs. The goal is to strengthen problem-solving intuition for interview-style questions and improve reasoning about real systems like dependency flows and service communication.

Impact:

Reduced dependency on middleware (LiteLLM)
